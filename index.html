Variable Fonts

Variable fonts enables many different variations of a typeface to be incorporated into a single file instead of having a separate font file for every width, weight or style. You can access all the variations contained in a given font file through CSS and a single @font-face reference.

Standard or Static Fonts

In the past if you haven’t used variable fonts you would have separate files for each individual fonts and what that font would represent like a specific width/weight/style combination. There would be a separate file for the font you were using like the name of the font such as: “Roboto Regular”, “Roboto Bold” and “Roboto Bold Italic”. This means that you would end up with 20 or 30 different font files to represent a complete typeface or even more.

With Variable fonts all styles can be contained in a single file and you can use CSS to access all the styles contained in a single font file without any worries about browser distortions. This is way more convenient than having a cluster of files just for you fonts. The advantage of choosing the variable font is that you have access to the entire range of weight, widths and styles which helps to design your site better, like changing the headings in different weights for readability or using a slightly narrower width for data-dense displays.

Variation Axis

Axis of variation describes the allowable range of the particular aspect of the typeface design. The weight axis describes how light or how bold the letterforms can be; The width axis describes how narrow or how wide they can be; the italic axis describes if italic letterforms are present and can be turned on or off accordingly. An axis can be a range or a binary choice. Weight can range to 1-999 and Italic might be 0 or 1 (off or on).

As defined in the specification, there are two kind of axes: registered and custom:

Registered axes are those that are most frequently encountered and common enough that the authors of the specification felt it was worth standardizing. The five currently registered axes are weight, width, slant, italic and optical size.

Custom axes are limitless: the typeface designer can define and scope any axis they like and are just required to give it a four-letter tag to identify it within the font file for format itself. You can use these four-letter tags in CSS to specify a point along that axis of variation.
